## Current CG Configuration

- `N <= 100`: `use_full_pool_small=True`, `full_pool_n=100`, complete generated column pool is solved, and metadata is `cg_status=full_pricing_converged`, `gap_type=Full-CG LP-IP gap`, `pricing_converged=True`.
- `N = 200` and `N = 500` targeted runs: strengthened budgeted CG uses `min_iters=20`, `max_iters=30`, `pricing_top_k=5`, incumbent injection enabled, Greedy/FIFO incumbents injected by default, and external incumbent schedules can be injected through `cg.incumbent_solutions`.
- Budgeted large-scale metadata uses `budgeted_stabilized` only if the last-5 relative LP improvement and pool gap criteria pass; otherwise it uses `budgeted_max_iter`. Large-scale results remain `Pool LP-IP gap` unless pricing truly converges.

## Warm Start / Incumbent Injection

- Explicit warm-start basis stabilization is not implemented.
- Incumbent column injection is implemented in `src/solvers/cg_solver.py`:
  - `convert_solution_to_columns`
  - `inject_incumbent_solution_columns`
  - internal Greedy/FIFO incumbent generation when `use_incumbent_injection=True`
- SIMOPS targeted dominance solves sequential first and injects the sequential selected columns into the SIMOPS run through `incumbent_solutions`.
